# About Project BOSC

Project BOSC reconstructs the public record behind a contested data-center
project in Lima, Ohio — built by a Delaware shell for an end user later confirmed
to be Google — and publishes it so anyone can check it. The record was kept
deliberately thin; reassembling it in the open, with provenance on every figure,
is the work.

## The method — four steps, every figure

<div class="callout callout--note">
  <p class="callout-title">Source → read → meaning → verify</p>
  <p>
    1. <strong>Source</strong> — a primary document (a deed, a permit, a fact sheet),
    cited by file and page.
    2. <strong>Read</strong> — read from the <em>image</em>, not the unreliable OCR text
    layer, into a typed, schema-validated record.
    3. <strong>Meaning</strong> — said plainly, with the load-bearing number called out.
    4. <strong>Verify</strong> — a link back to the source, so you never take our word for it.
  </p>
</div>

## The provenance promise

- **Every figure carries a source and an evidence tag.** `[verified]` is read from a
  cited record; `[inference]` is a labelled reading of it; `[open]` marks what's
  withheld or unresolved.
- **Inference is never dressed as fact.** Registered-agent and organizer overlaps are
  *common-control plumbing* — evidence of coordination, not a verdict on ownership.
- **Redactions are shown, not hidden.** Where a figure is sealed as a trade secret,
  the redaction itself is part of the record.
- **Nothing here is legal advice, and nothing here is a verdict.** Verify every figure
  against its cited source before quoting it.
